The Emacs text editor

Initial impressions

Initial Impressions using Emacs were pretty good. I was excited to start learning this program. At first I watched a few of those videos about Emacs on Moodle and then I googled some keybinds. I ran into my first issue, when I wanted to copy a piece of my code; the C+space mark hotkey didn't work. I had to look to google for some answers and it seems there is an alternative way to mark a location using c+@, that worked for me.

I ran into my second issue, when trying to figure out why my website didn't respond accordingly with the changes I made to my .css file. It seems I had accidentally created some other files with similar names and the .html file was reading them as the stylesheet for the page. Often the hotkeys and shortcuts are very useful, but sometimes stuff like that happens and you are left wondering what went wrong.

Great features

The editor allows for great customizability, add-ons, has great built in features and has a ton of resources online if you are looking for help. My favourite part about this editor is how beginner friendly it is. The f10 key and being able to see the hotkeys really helped me in the beginning of this exercise. I still have not figured out how to use my mouse to move my cursor around, but in the long term that might be a good thing, for it encourages the use of the useful hotkeys.

I personally think that the text editor is good, it has way more features and uses than notepad for example. I actually see myself using Emacs in the future. One thing that might make me a bit biased towards this text editor is that I started using this before using vim or Micro so the user interface and hotkeys are more intuituve.
